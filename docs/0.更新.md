## 0.1.4 修改

- 消除 `__init__.py` import 副作用：`GlobalTimeBase` 初始为 `None`，不再在导入时自动初始化
- 新增 `init()` 函数，用户需显式调用以初始化全局时基与终端输出
- 全面补全 `__all__` 公开接口，从 12 个扩展至 57 个，覆盖全部可用类/函数/常量
- 创建全局变量管理
- 创建 `MessageIO` 发布/订阅模式，支持多种输出并行（终端、文件、WebSocket、GUI），未完成
- 修复 `register.py` 中 `Field` 类的 `select` 字段 mutable default 问题
- 去除还未实现的report功能依赖的`reportlib`库，以紧急修复客户的依赖问题，未进行测试

## 0.1.3 修改

- 终端时间戳小数位数: 3 -> 5
- 终端时间戳长度修改为: 13(纯时间戳) + 3(中括号以及空格)，以匹配一些ssh返回的`\t`对齐
- 修改`SSH.exec_run()`方法，加入`sudo`、`password`参数，支持提权运行
- 添加`SSH.exec_run_sudo()`方法，支持提权运行，是`SSH.exec_run()`方法的简单包装
- `CmdRecord`类添加`stdout`、`stderr`属性，并在执行run方法时存储
- 提取`stdout`处理，添加到`SSH._cmd_output_process()`内部方法
- 添加`SSH.close_sub_channels`、`SSH.close_sub_sftp()`、`SSH.close_all()`方法，去除`SSH.__del__()`方法
- 规范`SSH`的`with`上下文管理
- 创建`metaclass.py`用于实现测试协议的统一管理，测试具体产品时的配置基类
- 为`SSH`、`Channel`、`SFTP`类添加最近完成命令返回的record缓存属性`.last_record:CmdRecord[str]`
- `Bits`类添加切片置位方法`set_bits() __setitem__() `
- `Bits`类添分割方法`split() split_iter() ` 
- `Bits`类添加基础比较方法：`__eq__(==)` `__ne__(!=)` `__le__(<=)` `__lt__(<)` `__gt__(>)` `__ge__(>=)`
- `Bits`类支持`bool`初始化
- `Bits`类添加添加`concat()`、`pop()`、`remove()`方法；`__setitem__()`支持了迭代赋值
- 添加`Bits._git_value()`
- `Bits`的`__getitem__()`与`__setitem__()`支持`[:]`全切
- 添加`BitView`类作为字段借用，与链接的`Bits`类同步数据
- 添加`Addr32`、`Addr64`类，是`Bits`类的简单封装与方法重载
- 添加`Field`类，主要作为`Register`类的附属
- 添加`Register`类，管理`Field`与`Addr32/Addr64`，可绑定读写寄存器的方法
- 修改`Bits`行为，支持0位宽，`pop`、`remove`等涉及到的方法用尽位宽后剩余`Bits(0,0)`，可以使用`Bits(None)`初始化得到
- 同步修改基础比较方法，`Bit(0,0) == Bits(None) == None`，尽管并不建议使用 `Bits(None) == None` ，会与`Bits(None) is None`恒为`False`混淆
- `Bits`添加`+`、`-`、`>>`、`<<`运算支持，包括右运算和自身运算
- `Bits`添加`|`、`&`、`^`运算支持，包括右运算和自身运算；以及`~`运算
- 实现了基本的`Serial`、`SerialConfig`类，进行了基本测试
- 实现了基本的`Telnet`、`TelnetConfig类`，验证`telnetlib3.sync`实现的可行性
- `Telnet`类实现了基本非阻塞收发、实现命令式`shel_run()`方法、实现了`shell_run_line_generator`行生成器方法
- `Channel`类的`run`方法对逐行输出使用`yield`生成器进行了解耦，提高了输出时间戳的实时性
- `Serial`拆分出`SerialShell`类专门用作串口类终端交互，并添加和修改了一系列方法
- `Serial`类作为基础串口操作，主要是对于bytes读写的方法，添加了一些基础处理方法
- `Telnet`拆分出`TelnetShell`类实现终端访问
- `remove_ansi`函数统一了`str`与`bytes`参数，并添加override
- `Bits`类补全四则运算实现：`*`、`*=`、`//`、`//=`，另外`/`等同于`//`、`/=`等同于`//=` 
- `Bits`类添加`@`、`@=`符号支持`Bits`拼接，分别是`Bits.concat()`方法和`Bits.append()`方法的简写
- 去除遗留的numpy转换函数
- `Bits`类添加max方法返回最大值Bits实例
- `Field`、`Register`类重写，移入`register.py`中，并添加`HasParent`基类与`RegGroup`类寄存器组
- `baseinfo.py`加入屏幕检测函数测试，为gui适配

# 项目结构
## 目前src树

```bash
src/autestoy
├── __init__.py              # 入口，暴露公开接口
├── export/                  # 导出与输出
│   ├── collect.py             # @collect 装饰器 + CollectObj
│   ├── markdown.py            # Markdown / Obsidian 导出
│   ├── messageio.py           # MessageIO 发布/订阅消息总线
│   ├── pdf.py                 # PDF 导出（占位）
│   ├── term.py                # 终端输出（时间戳、ANSI 颜色）
│   └── web.py                 # Web 界面（占位）
├── protocols/               # 通信协议
│   ├── jtag.py                # JTAG（占位）
│   ├── metaclass.py           # DUTConfiguratorBase 元编程基类
│   ├── serial.py              # 串口（Serial / SerialShell）
│   ├── ssh.py                 # SSH / Channel / SFTP
│   └── telnet.py              # Telnet（Telnet / TelnetShell）
└── tools/                   # 工具
    ├── ansi.py                # ANSI 转义序列工具
    ├── baseinfo.py            # 屏幕基础信息
    ├── control.py             # TrySeconds / ulog / get_line_from_head
    ├── datatype.py            # Bits / BitView / Addr32/64 / Packet
    ├── gui.py                 # DearPyGui（占位）
    ├── local.py               # 本地命令执行
    ├── record.py              # CmdRecord / CmdRecording / MetaRecord
    ├── register.py            # HasParent / Field / Register / RegGroup
    ├── result.py              # Result<T> 泛型包装
    └── timestamp.py           # Timestamp 时间戳
```

- [ ] `CollectObj` 引用泄漏：被 `@collect` 装饰的实例永久保留在全局列表中，无清理机制；`MarkdownExporter` 依赖此列表收集命令记录，耦合隐式

## 🟡 P1 — 建议修复

- [ ] `TermStyle` 的颜色配置为类变量，多线程/多测试用例间互相影响，建议改为实例化配置或 ContextVar
- [ ] `Timestamp.fmt` / `sw_utc` / `millis_width` 为类变量，修改会影响所有实例
- [ ] 自定义异常层次：全库使用内置 `RuntimeError`/`ValueError`，缺少 `AutestoyError` → `ConnectionError` / `AuthenticationError` / `PromptNotFoundError` 等
- [ ] `__all__` 缺少 `MessageBus`、`Message`、`MessageType`、`MessageSource`、`MessageTerminal`，用户无法通过 `import autestoy` 直接访问 MessageIO 发布/订阅功能

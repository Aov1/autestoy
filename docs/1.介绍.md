# 简介

autestoy（auto + test + toy）是一个面向嵌入式/硬件测试场景的 Python 自动化测试工具库，目前处于早期原型阶段。

## 设计理念

- **协议可移植**：协议层保持抽象，不引入具体产品细节。例如 SSH、Serial、Telnet 各为独立模块，不绑定特定设备型号。
- **贴近硬件思维**：数据类（如 `Bits`）的切片、拼接、进制显示等语义贴近 Verilog 语法，降低嵌入式工程师的使用门槛。
- **输出解耦**：通过 `MessageIO` 发布/订阅模式将协议执行与输出渲染分离，同一份测试脚本可同时输出到终端、文件、GUI 或 Web。
- **渐进可用**：优先完善一条完整的"连接 → 执行命令 → 记录结果 → 导出报告"链路，在此基础上逐步丰富多协议、多环境支持。

## v0.2.0 主要变更

### import 行为变更

**之前**：`import autestoy as att` 时会自动创建 `GlobalTimeBase`、绑定终端时基，并打印一行启动信息。

**之后**：import 是纯的，不产生任何副作用。用户需显式调用 `att.init()` 完成初始化：

```python
import autestoy as att
att.init()   # ← 创建时基、绑定终端、打印启动信息
```

> 如果你的旧代码依赖 import 时的自动初始化，升级到 v0.2.0 后请在脚本开头添加 `att.init()`。

### 接口完整暴露

`__all__` 从 12 个扩展至 57 个，现在可以通过 `att.XXX` 访问所有公开类与函数，不再需要深层 import。

### MessageIO 消息总线

新增 `autestoy/export/messageio.py`，提供发布/订阅模式解耦协议与输出。详情见 MessageIO 相关文档与示例。

# 简介

autestoy（auto + test + toy）是一个面向嵌入式/硬件测试场景的 Python 自动化测试工具库，目前处于早期原型阶段。

## 设计理念

- **协议可移植**：协议层保持抽象，不引入具体产品细节。例如 SSH、Serial、Telnet 各为独立模块，不绑定特定设备型号。
- **贴近硬件思维**：数据类（如 `Bits`）的切片、拼接、进制显示等语义贴近 Verilog 语法，降低嵌入式工程师的使用门槛。
- **输出解耦**：通过 `MessageIO` 发布/订阅模式将协议执行与输出渲染分离，同一份测试脚本可同时输出到终端、文件、GUI 或 Web。
- **渐进可用**：优先完善一条完整的"连接 → 执行命令 → 记录结果 → 导出报告"链路，在此基础上逐步丰富多协议、多环境支持。

## 模块总览

```
src/autestoy/
├── __init__.py      入口，暴露公开接口，提供 init() 初始化
├── export/          输出与导出
│   ├── term.py      终端输出（时间戳、ANSI 颜色）
│   ├── collect.py   实例收集装饰器 + CollectObj
│   ├── markdown.py  Markdown / Obsidian 导出
│   ├── messageio.py MessageIO 发布/订阅消息总线
│   ├── pdf.py       PDF 导出（占位）
│   └── web.py       Web 界面（占位）
├── protocols/       通信协议
│   ├── ssh.py       SSH / Channel / SFTP
│   ├── serial.py    串口（Serial / SerialShell）
│   ├── telnet.py    Telnet（Telnet / TelnetShell）
│   ├── jtag.py      JTAG（占位）
│   └── metaclass.py DUT 配置基类 + 动态方法注册
└── tools/           工具
    ├── ansi.py      ANSI 转义序列处理
    ├── baseinfo.py  屏幕基础信息
    ├── control.py   TrySeconds / ulog / get_line_from_head
    ├── datatype.py  Bits / BitView / Addr32/64 / Packet
    ├── gui.py       DearPyGui GUI（占位）
    ├── local.py     本地命令执行
    ├── record.py    CmdRecord / CmdRecording / MetaRecord
    ├── register.py  HasParent / Field / Register / RegGroup
    ├── result.py    Result<T> 泛型包装
    └── timestamp.py Timestamp 时间戳
```

## 未来规划

- DearPyGui 实时仪表盘

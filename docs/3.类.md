## 1.1 `ssh.py`

### 1.1.1 `class RemoteConfig`

SSH 远程连接配置。

```python
def __init__(
    self,
    user     : str,   # 用户名
    ip       : str,   # IP 地址
    password : str,   # 密码
    port     : int = 22,  # SSH 端口，默认 22
) -> None
```

默认名称为 `f"{user}@{ip}"`，影响终端输出样式：

```
[   0.000][1][root@192.168.100.188] $ ls
```

#### `set_name(self, name: str) -> Self`

设置自定义名称，覆盖默认 `user@ip`，支持链式调用。

```
conf.set_name("AMD7700")
# → [   0.000][1][AMD7700] $ ls
```

---

### 1.1.2 `class SSH`

SSH 主类，管理远程连接、命令执行、通道与 SFTP。

#### `__init__(self, remote_config, timeout=None, raise_when_timeout=True) -> None`

| 参数 | 类型 | 说明 |
|------|------|------|
| `remote_config` | `RemoteConfig` | 远程配置实例 |
| `timeout` | `float \| None` | 连接超时（秒），默认不启用 |
| `raise_when_timeout` | `bool` | 超时时是否抛出异常，默认 `True` |

```python
conf = RemoteConfig(user="root", ip="192.168.1.100", password="123")
dut = SSH(conf, timeout=20)
```

#### `is_connected(self) -> bool`

返回 SSH 连接是否存活。

#### `set_global_path(self, path: str) -> None`

设置全局路径。所有后续 `exec_run()` 命令均在 `cd {path} && exec {cmd}` 等效环境下执行。

#### `with_path(self, path: str) -> Self`

设置单次临时路径，仅对紧随的下一条命令生效。优先级高于 `global_path`。支持链式调用：

```python
dut.with_path("project/test").exec_run("ls")
```

#### `cd(self, path: str) -> CmdRecord[str]`

改变全局路径。实际执行 `cd {path} && pwd`，用返回的路径覆盖 `global_path`。路径不存在时不会更新。

#### `exec_run(self, cmd, sudo=False, password=None) -> CmdRecord[str]`

执行一条命令，返回 `CmdRecord[str]`。每次调用创建一个独立通道，不保持上下文。

| 参数 | 类型 | 说明 |
|------|------|------|
| `cmd` | `str` | 命令字符串 |
| `sudo` | `bool` | 是否以 sudo 提权运行 |
| `password` | `str \| None` | sudo 密码 |

```python
res = dut.exec_run("pwd")
# [   0.532] [1][DUT] $ pwd
# [   0.589] /home
```

#### `exec_run_sudo(self, cmd, password) -> CmdRecord[str]`

等价于 `exec_run(cmd, sudo=True, password=password)`。

#### `exec_run_lines(self, *cmds) -> list[CmdRecord[str]]`

顺序执行多条命令。支持多行字符串、列表、元组、迭代器：

```python
dut.exec_run_lines("""
./step1.sh
./step2.sh
./step3.sh
""")
dut.exec_run_lines("cmd1", "cmd2", "cmd3")
dut.exec_run_lines(["a.sh", "b.sh"])
```

#### `long_running(self, cmd, start_task=True) -> CmdRecording[str]`

多线程后台执行命令，不阻塞主线程。返回 `CmdRecording[str]`（继承自 `CmdRecord`，增加 FIFO 与线程控制）。

| 参数 | 类型 | 说明 |
|------|------|------|
| `cmd` | `str` | 命令 |
| `start_task` | `bool` | 是否立即启动线程，默认 `True` |

```python
task = dut.long_running("tail -f /var/log/syslog")
# ... 主线程执行其他操作 ...
task.task_kill()
```

#### `long_running_list(self, *cmds, start_task=True) -> list[CmdRecording[str]]`

同时创建多个后台任务：

```python
tasks = dut.long_running_list("cmd1", "cmd2", "cmd3", start_task=True)
```

#### `create_channel(self, name=None, prompt_pattern=None, show_welcome_info=False, insert_cmd=None) -> Channel`

创建交互式 SSH 通道。通道内保持目录、变量等上下文。

| 参数 | 类型 | 说明 |
|------|------|------|
| `name` | `str \| None` | 通道名称，默认 `ch_n` |
| `prompt_pattern` | `str \| None` | 终端提示符正则，默认匹配常见 prompt |
| `show_welcome_info` | `bool` | 是否显示通道创建时的输出 |
| `insert_cmd` | `str \| None` | 创建后立即执行的命令（如 `"bash"` 切换 shell） |

#### `create_ftp(self) -> SFTP`

创建 SFTP 文件传输实例。

#### `close_all(self) -> None`

关闭所有子通道、SFTP 以及 SSH 连接本身。`with` 语句中自动调用。

#### `kill(self, pid) -> CmdRecord[str]`

杀死远程服务器上的进程。

---

### 1.1.3 `class Channel`

SSH 交互式通道，保持命令上下文和多行历史。

#### `__init__(self, ssh, name=None, prompt_pattern=None, show_welcome_info=False, insert_cmd=None)`

一般通过 `SSH.create_channel()` 创建。

初始化时会自动检测终端提示符，3 秒内未匹配则自动发送 `"bash\n"` 切换到 bash 环境。

#### `set_name(self, name: str) -> None`

重置通道名。

#### `run(self, cmd: str) -> CmdRecord[str]`

在通道中执行一条命令。保持目录和 shell 状态：

```python
ch.run("cd /home/project")
ch.run("pwd")   # /home/project
```

#### `run_lines(self, *cmds) -> list[CmdRecord[str]]`

按顺序执行多条命令，支持多行字符串、列表等。

---

### 1.1.4 `class SFTP`

SFTP 文件传输，封装 paramiko 的 SFTPClient。方法签名与 paramiko 一致，返回值包装为 `CmdRecord`。

主要方法：

```python
sftp = ssh.create_ftp()
sftp.put(local_path, remote_path)
sftp.get(remote_path, local_path)
sftp.listdir(remote_path)        # → list[str]
sftp.listdir_attr(remote_path)   # → list[SFTPAttributes]
sftp.mkdir(remote_path)
sftp.remove(remote_path)
sftp.rename(old, new)
sftp.stat(remote_path)
sftp.chmod(remote_path, mode)
sftp.chown(remote_path, uid, gid)
sftp.exists(remote_path)         # 通过自动 mkdir 支持
```

## 1.2 `serial.py`

### 1.2.1 `class SerialConfig`

串口配置。

```python
def __init__(
    self,
    port     : str | None = None,
    baudrate : int = 115200,
    bytesize : int = 8,
    parity   : str = "N",
    stopbits : float = 1,
    timeout  : float | None = 0.05,
    ...
)
```

#### `set_name(self, name: str) -> Self`

自定义名称，默认 `f"{port}@{baudrate}"`。

### 1.2.2 `class Serial`

基础串口操作，面向 bytes 读写。

```python
uart = Serial(SerialConfig("/dev/ttyUSB0", 115200))
uart.send(b"AT\r\n")
data = uart.recv_no_block(1024)
```

**主要方法：**

| 方法 | 说明 |
|------|------|
| `send(data: bytes)` | 发送数据 |
| `recv_no_block(size=1) -> bytes` | 非阻塞读取 |
| `recv_until(re_delimiter: bytes) -> tuple[bytes, Match]` | 读取直到正则匹配 |
| `readline_generator() -> Generator[(bool, str)]` | 逐行读取生成器 |
| `recv_bytes_generator() -> Generator[bytes \| None]` | 逐字节读取生成器 |

### 1.2.3 `class SerialShell(Serial)`

串口终端交互模式，支持命令执行、登录、sudo 提权。

```python
shell = SerialShell(SerialConfig("/dev/ttyUSB0", 115200))
shell.shell_run("uname -a")
shell.shell_run_sudo("systemctl restart sshd", password="root")
```

#### `shell_run(self, cmd) -> CmdRecord[str]`

发送命令并捕获输出，通过终端提示符判断命令结束。

#### `shell_run_sudo(self, cmd, password) -> CmdRecord[str]`

sudo 提权执行单条命令，执行后清除 sudo 缓存。

#### `shell_run_lines(self, *cmds) -> list[CmdRecord[str]]`

顺序执行多条命令。

#### `login(self, ...) -> str`

自动登录串口终端（用户名/密码交互）。

## 1.5 `metaclass.py`

### 1.5.1 `Protocol`

### 1.5.2 `class DUTConfiguratorBase`

DUT（Device Under Test）配置基类，用于统一管理多协议实例。

```python
class MyDevice(DUTConfiguratorBase):
    def __init__(self):
        super().__init__(
            SSH(RemoteConfig(user="root", ip="192.168.1.100", password="xxx")),
            Serial(SerialConfig("/dev/ttyUSB0", 115200)),
        )

dut = MyDevice()
ssh_list = dut.get_protocol(SSH)          # 获取所有 SSH 实例
uart = dut.get_from_name(Serial, "ttyUSB0") # 按名称获取
if SSH in dut:                            # 判断是否存在
    ...
dut[SSH][0].exec_run("ls")               # 下标访问
```

#### `get_protocol(self, protocol_type) -> list[P]`

获取指定类型的所有协议实例。

#### `get_from_name(self, protocol_type, name) -> P`

通过协议类型和名称获取实例。

#### `add_dynamic_method(self, name, func)`

注册动态方法，通过 `self.name()` 调用。

#### `__getitem__(protocol_type) -> list[P]`

下标语法获取协议实例。

#### `__contains__(protocol_type_or_name) -> bool`

`in` 语法判断协议是否存在。

## 2.4 `messageio.py`

发布/订阅消息总线，解耦协议输出与输出渲染。

### 2.4.1 `class MessageType(StrEnum)`

消息类型枚举：

- `CMD_START`, `CMD_OUTPUT`, `CMD_END`, `CMD_ERROR`
- `CONNECT`, `DISCONNECT`, `LOG`
- `SCRIPT_START`, `SCRIPT_END`

### 2.4.2 `class Message`

消息体：

```python
@dataclass
class Message:
    type: MessageType      # 消息类型
    source: str            # 来源标识（如 "SSH:dut1"）
    timestamp: float       # 时间戳
    data: dict             # 附加数据
```

### 2.4.3 `class MessageIO`

消息总线（类方法）。

| 方法 | 说明 |
|------|------|
| `subscribe(msg_type, callback)` | 订阅消息，返回取消订阅函数 |
| `publish(message)` | 发布消息 |
| `reset()` | 清理所有订阅 |

Handler 示例：

```python
class TerminalHandler:
    def __init__(self):
        MessageIO.subscribe(MessageType.CMD_OUTPUT, self._on_output)
    def _on_output(self, msg: Message):
        sys.stdout.write(f"{msg.data['line']}\n")
```

## 3.1 `record.py`

### 3.1.1 `class CmdRecord(Generic[T])`

命令执行记录，供所有执行方法返回使用。

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | `int` | 命令自增 ID |
| `cmd` | `str` | 执行的命令 |
| `prompt` | `str` | 终端提示符 |
| `start_time` | `Timestamp` | 开始时间 |
| `end_time` | `Timestamp \| None` | 结束时间 |
| `run_time` | `float \| None` | 运行时长（秒） |
| `result` | `list[(Timestamp, Result[T])]` | 输出行列表 |
| `exit_code` | `int \| None` | 退出码 |
| `stdin` | `ChannelStdinFile \| None` | 标准输入（仅 SSH） |
| `stdout` | `ChannelFile \| None` | 标准输出（仅 SSH） |
| `stderr` | `ChannelStderrFile \| None` | 标准错误（仅 SSH） |

**方法：**

| 方法 | 说明 |
|------|------|
| `get_result() -> list[str]` | 获取输出文本列表（去除 ANSI） |
| `get_result_string() -> str` | 合并输出为单一字符串 |
| `get_result_iter() -> Iterator[str]` | 输出迭代器 |
| `search(pattern) -> Match \| None` | 正则搜索，首个匹配 |
| `search_all(pattern) -> Iterator[Match]` | 正则搜索，所有匹配 |
| `cut_fields(*fields, re_delimiter) -> list[list[str]]` | 按分隔符切割字段 |
| `cut_characters(*ranges) -> list[list[str]]` | 按字符位置切割 |
| `record_end()` | 手动记录结束时间 |
| `get_run_time() -> float \| None` | 获取运行时长 |

```python
record = dut.exec_run("ls -l")
# 字段切割
fields = record.cut_fields(5, 9, re_delimiter=r"\s+")
# 字符切割
chars = record.cut_characters((1, 10), (50, 60))
# 正则搜索
if match := record.search(r"error|fail"):
    print(match.group(0))
```

### 3.1.2 `class CmdRecording(CmdRecord[T])`

多线程命令记录，增加 FIFO 队列与线程控制。

| 方法 | 说明 |
|------|------|

| `task_join()` | 等待线程结束 |
| `task_kill()` | 终止线程（发送 Ctrl+C + 设置停止事件） |
| `is_task_alive() -> bool` | 线程是否存活 |
| `get_once() -> str \| None` | 读一次 FIFO |
| `clean_fifo()` | 清空 FIFO |
| `force_get_fifo(times, timeout, raise) -> list[str]` | 强制获取 N 行 FIFO |
| `search_next_line(pattern) -> tuple[str, Match] \| None` | FIFO 逐行正则匹配 |
| `find_next_line(pattern) -> tuple[str, list] \| None` | FIFO 逐行查找 |
| `fifo_wait(pattern, timeout) -> tuple[list[str], bool]` | 等待 FIFO 中出现匹配行 |

### 3.1.3 `class MetaRecord(Generic[T])`

## 3.6 `datatype.py`

### 3.6.1 `class Bits`

类 Verilog 位宽数据类型。

**构造：**

```python
Bits(value, width=None)
```

| value 类型 | 说明 |
|-----------|------|
| `None` | 空值，位宽 0 |
| `int` | 整数值，需配合 `width` 使用 |
| `bool` | 布尔值 |
| `str` | 支持 `"8'd255"`、`"16'hFF00"`、`"32'o377"`、`"2'b1010"` 等 Verilog 风格 |
| `Bits` | 复制构造 |
| `Iterable` | 拼接多个值 |
| `(value, width)` tuple | 值 + 位宽 |

**常用属性/方法：**

| 成员 | 说明 |
|------|------|
| `.value` | 整数值 |
| `.width` | 位宽 |
| `.max()` | 返回全 1 的最大值 |
| `bits[7:0]` | 切片读取（高位/低位自动适配） |
| `bits[7:0] = 0xFF` | 切片赋值 |
| `a @ b` | 拼接（等价于 `concat`） |
| `~bits` | 按位取反 |
| `.split(n)` | 拆分为 n 个 Bits |
| `.split_iter(n)` | 拆分迭代器 |
| `.append(bits)` | 低位追加 |
| `.concat(bits)` | 高位拼接 |
| `.pop()` | 弹出最低位 |
| `.remove()` | 移除最高位 |
| `.set_bits(range, value)` | 设置字段值 |
| `fmt(base, digit_sep, ...)` | 自定义格式化 |

**类配置（全局生效）：**

```python
Bits.set_verilog_like(True)          # 使用 Verilog 风格显示
Bits.set_str_type(16)                # 默认进制
Bits.set_digit_separator("_", ...)   # 数字分隔符
Bits.set_upper_hex_digits(True)      # 大写十六进制
```

**格式化：**

```python
b = Bits(0xAB, 8)
str(b)            # "0xab"
f"{b:b}"          # "10101011"
f"{b:4b}"         # "1010_1011"
f"{b:16X}"        # "0XAB"
f"{b:4d}"         # "0171"
f"{b:4v}"         # "8'hab"
```

### 3.6.2 `class BitView`

Bits 的字段借用视图，对 BitView 的修改会同步到原始 Bits：

```python
reg = Bits(0x12345678, 32)

view.value = 0xA5A5
print(reg)                    # 低 16 位已更改
```

### 3.7.2 `class Field(HasParent)`

### 3.7.3 `class Register(HasParent)`

寄存器，包含多个 Field。

| 属性 | 类型 | 说明 |
|------|------|------|
| `address` | `Addr32 \| Addr64 \| Bits` | 地址 |
| `info` | `str` | 描述 |
| `.offset` | `Bits` | 相对于基地址的偏移（由 RegGroup 自动计算） |
| `.default()` | `tuple[Bits, Bits]` | 返回 (default_pattern, default_value) |

```python
@dataclass(slots=True)
class GLB_CTL(Register):
    address: Addr32 = Addr32(0x1234_5678, 32)
    MODE: Field = Field(bit_range=(3, 0), default=Bits(0, 4))
    SPEED_SEL: Field = Field(bit_range=(15, 4), default=Bits(0x800, 12))
```

### 3.7.4 `class RegGroup(HasParent)`

寄存器组，可嵌套。

| 属性 | 类型 | 说明 |
|------|------|------|
| `base_address` | `Addr32 \| Addr64 \| Bits` | 基地址 |
| `end_address` | `Addr32 \| Addr64 \| Bits` | 结束地址 |
| `range` | `tuple` | (base, end) |

```python
@dataclass(slots=True)
class STM32_PERIPH(RegGroup):
    base_address: Addr32 = Addr32(0x4000_0000, 32)
    end_address: Addr32 = Addr32(0x4000_FFFF, 32)
    GLB_CTL: GLB_CTL = GLB_CTL()    # Register 实例

```

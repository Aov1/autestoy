## 1.1 `ssh.py`

### 1.1.1 `class RemoteConfig`

配置ssh连接，只是方便写好默认配置，然后快速导入初始化。

#### 1.1.1.1 `__init__(self,...)->None`

```python
def __init__(self,
	user     : str ,     # 用户名，字符串
	ip       : str ,     # ip地址，字符串
	password : str ,     # 密码，字符串
	port     : int = 22  # 端口号，int，默认22
) -> None
```

默认创建名称为`f"{user}@{ip}"`

影响终端输出样式，例如 `user="root"，ip="192.168.100.188"`

运行时输出：`[   0.000][1][root@192.168.100.188]~$ ls`

#### 1.1.1.2 `set_name(self,name:str)->Self`

```python
def set_name(self,
	name : str  # 设置名称，覆盖原始名称f"{user}@{ip}"
) -> Self       # 返回自身，所以可以链式调用（虽然暂时没啥用）
```

例如修改名称 `conf.set_name("AMD7700")`

运行时i输出：`[   0.000][1][AMD7700]~$ ls`

### 1.1.2 `SSH`类

#### 1.1.2.1 `__init__(self,...)->None`

SSH类初始化。

```python
def __init__(self, 
	remote_config      : RemoteConfig,       # 传入远程配置实例
	timeout            : float | None = None # 设置超时，默认不启用超时
	raise_when_timeout : bool = True         # 超时是否抛出异常 add 0.1.2
) -> None:
```

例子：
```python
amd7700_linux = RemoteConfig(
	user = 'root',
	ip = '192.168.100.155',
	password = '123',
).set_name("AMD7700")

dut = SSH(and7700_linux , 20) # 应用amd7700_linux的配置，设置20s超时

```
> [!warning]
> 0.1.1版本超时后默认不会抛出异常，仅在终端输出warning。目的是配合is_connected(self)方法作连接失败的错误处理。规划后续增加开关控制超时行为。
> 0.1.2版本以后，添加raise_when_timeout:bool参数，控制超时是否抛出异常


#### 1.1.2.2 `is_connected(self)->bool`

返回ssh是否连接成功

#### 1.1.2.3 `set_global_path(self ,path : str ) -> None`

设置全局路径，所有后续命令都会在该路径下执行

#### 1.1.2.4 `with_path(self ,path : str ) -> Self`

设置当前命令路径，只生效一次，可以链式调用

```python
remote_config = RemoteConfig(...)
pc = SSH(remote_config)
pc.with_path("project/test").exec_run("ls") # 基于默认路径（一般是home）进行访问
pc.with_path("/deta/data/com.termux/files/home/project/test").exec_run("ls") # 基于完整路径访问
```

实现方式是在cmd前嵌入cd命令，等效于 `f"cd {temp_path} && {cmd}"` 。所以可以实现从默认路径相对访问和完整路径访问其他路径

在任何ssh的run方法执行后清除（其实是`self._path_process()`处理后清除），优先级高于global_path

#### 1.1.2.5 `cd(self ,path : str ) -> None`

改变当前路径，作用于global_path，但是受到with_path方法维护的temp_path的影响
当temp_path非空执行cd后清除temp_path，优先级高于global_path
不管被哪个路径影响，最后根据运行返回的路径覆盖global_path。
路径错误或不存在不会对gloabl_path赋值。

#### 1.1.2.6 `exec_run(self, cmd : str ) -> CmdRecord[str]`

exec_run，执行命令，返回输出信息记录类CmdRecord

```python
remote_config = RemoteConfig(...).set_name("HUAWEI MATEPAD 12.2")
remote_pc = SSH(remote_config)
res = remote_pc.exec_run('pwd')
#[   0.532] [1]:[HUAWEI MATEPAD 12.2] $ pwd
#[   0.589] /data/data/com.termux/files/home
# 使用返回的CmdRecord进行数据校验、处理等
```

实现方式为exec_command方法，每次执行都相当于开启一个新的通道，因此没有上下文保持（例如cd到新的路径后，在下一个exec_run中又进入到默认路径)

SSH类实现了with_path、set_global_path和cd方法，用于exec_run设置临时路径和全局路径

CmdRecord提供了诸如记录时间戳、获取运行时间、格式化输出，正则表达式匹配等方法，方便进行数据校验与处理。

#### 1.1.2.7 `exec_run_lines(self, *cmd : str | Iterable[str]) -> list[CmdRecord[str]]`

基于exec_run，可以按顺序执行多条命令。

空行会自动跳过

支持：
- 带有换行的单个多行命令字符串，将解析为多个命令
- 逗号分隔的多个命令字符串
- 可迭代对象（list、tuple、iter等）存储的多个命令字符串
- 以上三种形式的混合，不可嵌套

```python
# 从不知名文档中复制的流程
dut.exec_run_lines('''
./script1.sh # first step:xxxx
./script2.sh # sec step:xxxx
./script2.sh # last step:xxxx
''')
# 解包传参
dut.exec_run_lines('./script.sh','./script2.sh','./script3.sh',...)
# list
dut.exec_run_lines(['./script1.sh','./script2.sh','./script3.sh',...])
# tuple
dut.exec_run_lines(('./script1.sh','./script2.sh','./script3.sh',...))
```

#### 1.1.2.8 `long_running(self,...) -> CmdRecording[str]`

```python
def long_running(self,
	cmd        : str         , # 需要执行的命令
  # timeout    : float = 0.5 , # 任务start后的延时，鸡肋，目前dev分支已经去除，只存在于0.1.1中
	start_task : bool = True , # 控制是否立即执行，预计0.1.2合并，用于自定义控制
) -> CmdRecording[str]
```

对于命令创建一个线程执行，返回一个CmdRecording记录。不会等待命令结束。

- 需要等待命令结束：

```python
res = dut.long_running('dmesg -w')
# CmdRecording.long_running_task 保存了threading库的Thread类，可以使用其方法操作运行的线程
res.task_join() # 等待线程结束
```
- 需要提前终止线程：
```python
res = dut.long_running('dmesg -w')
do_something()
res.task_kill() # 终止线程
# 等同于:
res.stdin.write('\x03') # 向exec_command返回的远程stdin写\x03
res.stop_event.set()    # 运行threading库Event类实例.set()方法，向线程传递终止事件

# 断言线程是否被终止
assert res.is_task_alive() == False
```
- 多个long_running同时执行：
```python
res1 = dut.long_running('./task_1.sh',start_task = False)
res2 = dut.long_running('./task_2.sh',start_task = False)
res3 = dut.long_running('./task_3.sh',start_task = False)

res1.task_start()
res2.task_start()
res3.task_start()
```

CmdRecording继承于CmdRecord，拥有基类方法的同时具有专门处理多线程task的方法，例如结束线程、输出fifo，fifo单次正则匹配等。

#### 1.1.2.9 `long_running_list(self, ... ) -> list[CmdRecording[str]]`


```python
def long_running_list(self, 
	*cmds      : str | Iterable[str] , # 可接受多个cmd，每个cmd可以为str或其他可迭代对象 
	start_task : bool = True           # 是否立即运行
) -> list[CmdRecording[str]]:

```

对于long_running的包装，可以传入 多个令字符串、命令列表、命令元组、命令迭代器

> [!warning] 
> 此方法0.1.1暂未合并；计划0.1.2合并

```python
dut.long_running_list('''
long_running_cmd1
long_running_cmd2
long_running_cmd3
''')
dut.long_running_list('long_running_cmd1','long_running_cmd2','long_running_cmd3',...)
dut.long_running_list(['long_running_cmd1','long_running_cmd2','long_running_cmd3',...])
dut.long_running_list(('long_running_cmd1','long_running_cmd2','long_running_cmd3',...))
# 同时start
long_running_commands_iter = gen() # 设想这个命令提供了100个命令
# 创建所有任务，但不执行
res = dut.long_running_list(
	*long_running_commands_iter,
	start_task = False
)
# 同时(近似)开始执行
list(map(lambda x:x.task_start() , res )) # list隐式迭代
# 等待全部结束
list(map(lambda x:x.task_start() , res )) # list隐式迭代
```

### 1.1.3 `Channel`类

#### 1.1.3.1 `__init__(self) -> None`

```python
def __init__(self,
	ssh               : SSH               , # Channel是ssh连接下的一个通道，因此与要一个ssh实例
	name              : str | None = None , # 通道的名称，不设置时使用 ch_n
	prompt_pattern    : str | None = None , # Channel需要捕获终端提示符，以确定命令是否执行完毕
	show_welcome_info : bool = False      , # 是否显示Channel初始化时的输出（目前没有嵌入时间戳）
	insert_cmd        : str | None = None , # 在初始化时插入的命令
) -> None:
```
初始化Channel，一般直接使用`SSH.create_channel(...)`

目前默认的终端提示符捕获只能捕获常见的提示符，如果终端使用花里胡哨的提示符，例如zsh

会在内部3s计时超时后自动更换为bash环境（发送bash命令）

#### 1.1.3.2 `set_name(self,name:str)->None`

重置Channel的名称，用于终端显示和导出

#### 1.1.3.3 `run(self,cmd:str)->CmdRecord[str]`

在Channel下执行命令。由于在同一通道下执行，所以可以有上下文保持。

且由于保持同一通道，理论上速度比`SSH.exec_run(...)`要块上一些

```python
dut = SSH(RemoteConfig(...))
dut_ch1 = dut.create_channel()
res = dut_ch1.run('./script.sh')
if 'Fail' in res:
	ulog('script Fail')
	dp_something()
```

#### 1.1.3.4 `run_lines(self, *cmds: Iterable[str] | str) -> list[CmdRecord[str]]`

按顺序运行多行命令
```python
dut = SSH(RemoteConfig(...))
dut_ch1 = dut.create_channel()
# 单字符串多行命令
res = dut_ch1.run_lines("""
	./script1.sh
	./script2.sh
	./script3.sh
""")
for each_record in res:
	if 'Fail' in each_record:
		do_something()
# 多个命令
res = dut_ch1.run_lines('cmd1','cmd2','cmd3',...)
# 可迭代对象传入命令
res = dut_ch1.run_lines(['cmd1','cmd2','cmd3',...])
# 诸如此类
```

### 3.1.1 `CmdRecord`类

主要用于记录与处理命令的返回值

涉及方法：
```python
SSH.exec_run()       -> CmdRecord[str]
SSH.exec_run_lines() -> list[CmdRecord[str]]
Channel.run()        -> CmdRecord[str]
Channel.run_lines()  -> list[CmdRecord[str]]
SFTP.xxxx()          -> CmdRecord[str|None]
```

#### 3.1.1.1 `get_result(self) -> list[str]`

获取命令的输出结果，去除Result包装，去除ansi转义，得到纯字符串

对于非字符串的结果尝试转换为字符串

#### 3.1.1.2 `get_result_string(self) -> str`

获取命令的输出结果，将多行合并为一个字符串，通过哦`\n`连接每行

去除Result包装，去除ansi转义

#### 3.1.1.3 `get_result_iter(self) -> Iterator[str]`

获取命令输出的迭代器，去除ansi转义，去除Result包装。

适用于长输出

#### 3.1.1.4 `search(self,re_string:str)->re.Match[str]|None`

搜索命令输出结果，匹配到一个re_string立即返回，未匹配返回None

支持正则，返回的Match实例就是re库的Match类，使用方法一致

```python
dut = SSH(...)
res = dut.exec_run('ls')
if name:=res.search(r'log[\d]+\.txt'):
	# 会匹配如 log123.txt、log2.txt 等
	do_something_use(name.group())  # name.group() 获取匹配字符
```


#### 3.1.1.5 `search_all(self,re_string:str)->Iterator[re.Match[str]]`

搜索命令的输出结果，匹配所有符合描述的内容，返回迭代器，迭代器子类型还是re.match

```python
dut = SSH(...)
res = dut.exec_run('cat log.txt') # 查看log
# 匹配所有以addr开头的地址字符串
# 例如 addr : 0xABCD1234 ； addr:0xdeadbeef 
hex_iter = res.search_all(r'addr\s*:\s*(0x[0-9a-fA-F]{1:8})') 

got_addr = [] # 地址收集list
for each_hex in hex_iter:
	# 取出正则匹配的第一部分，即hex的内容
	# 转换为Bits类
	got_addr.append(Bits(each_hex.group(1),32)) 
	
```

#### 3.1.1.6 `cut_fields(self,...) -> list[list[str]]`
```python
def cut_fields(self,
	*fields       : int        , # 多个字段组
	re_delimiter  : str = r"\t", # 分隔字符，支持正则

```
类似cut命令的字段分隔，处理输出的内容，例如ps输出：

```bash
   5490 pts/0    00:00:00 zsh
   5500 pts/0    00:00:00 zsh
```
```python
res = dut_ch1.run('ps')
```

### 3.1.2 `CmdRecording`类

CmdRecording类继承自CmdRecord，在CmdRecord原有方法的基础上添加了多线程处理相关方法。

CmdRecording专门记录`SSH.long_running()/SSH.long_running_list()`此类多线程命令

与基类相比新加入的方法有：

```python
CmdRecording.is_task_alive(self)->bool # 线程是否存活
CmdRecording.task_start(self)->None    # 线程启动
CmdRecording.task_join(self)->None     # 等待线程结束（对于非循环命令）
CmdRecording.task_kill(self)->None     # 强制结束命令，退出线程
CmdRecording.get_once(self)->None|str  # 读一次fifo，可能为空
CmdRecording.clean_fifo(self)->None    # 清除fifo
CmdRecording.force_get_fifo(self,times=1,timeout=2,raise_when_timmeout=False)->list[str] 
# 强制获取fifo n次，一定会等待fifo中有值再读出，可设置超时（=0关闭），可控制超时是否抛出异常
CmdRecording.search_next_line(self,pattern,flag)->tuple[str,re.Match[str]|None]
# 正则匹配一次fifo，返回当前fifo和匹配结果，匹配结果只包含第一次
CmdRecording.find_next_line(self,pattern,flag)->tuple[str,list[str|None]]
# 正则查找一次fifo，返回当前fifo和匹配结果列表，列表包含所有匹配结果
CmdRecording.fifo_wait(self,re_string,timeout)->None|list[str]
# 等待fifo中出现匹配的正则表达式的内容，可设置超时，返回这个过程中fifo值的列表和是否成功。
```
